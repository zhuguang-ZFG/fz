## goal

Replace G4 placeholder with structured OTA evidence validation (Memfault/RainMaker style multi-round checklist), wired into `release_gate.py`.

## requirements

1. Create `release/g4_ota.template.yaml` with items e.g.:
   - ota.enabled_in_product
   - ota.old_to_new_success (count >= 1 or note)
   - ota.version_matches_artifact
   - ota.failure_recovery_documented
   - ota.usb_fallback_ok
   Each item: id, title, result (pass|fail|skip|na), note, evidence

2. Create `scripts/g4_ota.py` with `validate_g4_evidence(path, features) -> (status, report)`:
   - If `features.ota` is false → caller should skip (gate already does)
   - If ota true: required items cannot be bare skip without note containing "waive"
   - any fail → status fail
   - Mirror style of `scripts/g3_evidence.py` (simple YAML parse if no PyYAML)

3. Wire `release_gate.py`:
   - Add `--g4-evidence` Path optional
   - Replace `run_g4_placeholder` with real function:
     - ota false → skipped_no_ota
     - ota true, no evidence → unknown
     - ota true, evidence → validate

4. Unit tests in `scripts/test_g4_ota.py` (unittest), at least:
   - ota false not tested here (gate level)
   - sample pass when ota items pass
   - fail when required fail

5. Sample `release/g4_ota.dev-sample-pass.yaml` for tooling
